2.2 了解 Oracle Application Express

Oracle应用程序 Express 是一个集中部署的声明式开发环境，用于开发和部署以数据库为中心的Web应用程序。

2.2.1 什么是Oracle Application Express??

Oracle Application Express是一个集中部署的声明式开发环境，用于开发和部署以数据库为中心的Web应用程序。 借助用户界面主题，导航控件，表单处理程序和灵活的报表等内置功能，Oracle Application Express加快了应用程序开发过程。

Application Express引擎根据数据库表中存储的数据实时呈现应用程序。 创建或扩展应用程序时，Oracle Application Express会创建或修改存储在数据库表中的元数据。 运行应用程序时，Application Express引擎然后读取元数据并显示该应用程序。

要在应用程序中提供有状态行为，Oracle Application Express可以透明地管理数据库中的会话状态。 应用程序开发人员可以使用简单的文本替换标记以及标准的SQL绑定变量语法来获取和设置会话状态。

2.2.2 Oracle Application Express如何工作

Oracle Application Express随Oracle数据库一起安装，并由存放在表里的数据和PL/SQL代码组成。无论您运行Oracle Application Express开发环境还是运行使用Oracle Application Express构建的应用程序，过程都是相同的。您的浏览器发送一个URL请求，该请求被转换为相应的Oracle Application Express PL/SQL调用。数据库处理完PL/SQL之后，结果将以HTML的形式转发回您的浏览器。每当您请求或提交页面时，都会发生此循环。

Oracle Application Express不使用专用数据库连接。相反，每个请求都是通过新的数据库会话发出的，消耗的CPU资源最少。应用程序会话状态由Application Express引擎在数据库表中管理。

在后台， Application Express引擎生成并处理页面。 Application Express引擎还承担以下工作:

  • 会话状态管理

  • 验证服务

  • 授权服务

  • 页面跳转控制

  • 校验处理

2.2.3 关于运行时环境与完整开发环境之间的差异

安装Oracle Application Express时，可以安装两种不同的环境:

  • 运行时环境. 在 运行时环境中，用户可以运行应用程序，但不能对其进行修改。 选择此选项可用于生产实施。

  • 完整开发环境. 在 完整的开发环境中，用户可以开发，修改，运行和删除Application Express应用程序。 此选项提供了本文档中描述的Oracle Application Express环境的完整功能。
